# Chrome Web Store Release

This checklist prepares Swan for an unlisted Chrome Web Store beta. It keeps the v0 model unchanged: users install the extension, then configure their own ElevenLabs and optional Twilio credentials.

## Release channel

- Start with **Unlisted** visibility.
- Use public visibility only after the unlisted beta installs cleanly and the provider setup flow is proven with real users.
- Keep source-loaded installation documented for contributors and users who want to audit or modify the extension.

## Build package

Run the release checks from a clean checkout:

```bash
npm run test
npm run typecheck
npm run build:store
npm run zip:store
npm run docs:build
git diff --check
```

Upload the Chrome MV3 zip generated by WXT. Do not upload the source checkout, `node_modules`, local `config.yaml`, or docs build output.

Use `build:store` and `zip:store` for Chrome Web Store packages. These commands remove any generated `public/swan-bootstrap.json` before building so local `config.yaml` values cannot be bundled into the public release.

Before upload, inspect `output/chrome-mv3/manifest.json` and confirm:

- Manifest version is 3.
- The extension has 16, 32, 48, and 128 pixel icons.
- Permissions are limited to `storage` and `webNavigation`.
- Host permissions are limited to ElevenLabs and Twilio API endpoints.
- `swan-bootstrap.json` does not contain private provider credentials for a public release build.

## Store listing

Use a concise listing that avoids clinical claims.

Suggested short description:

```text
Adult-site urge intervention with AI phone calls.
```

Suggested longer description:

```text
Swan helps interrupt adult-site urge loops in the browser. Configure the domains you want to track, connect your own ElevenLabs voice agent, and Swan can call you at the moment of urge. Optional SMS alerts can be enabled with your own Twilio account.

Swan is recovery-support software, not medical advice, therapy, or clinical treatment. It does not run a Swan-hosted backend, inspect page content, classify images or videos, install DNS rules, or block at the operating-system level.
```

Store assets to prepare:

- 128x128 extension icon.
- At least one 1280x800 or 640x400 screenshot.
- 440x280 small promotional tile.
- Optional additional screenshots for settings, provider setup, domain rules, and intervention page.

## Privacy and permission declarations

Use the hosted docs privacy page as the privacy policy URL:

```text
https://dannylee1020.github.io/swan/privacy
```

Declare the data Swan stores or handles:

- Phone number.
- Authentication credentials for user-configured providers.
- User-configured domain rules.
- Extension activity logs for detected events.

Permission justifications:

- `storage`: stores local settings, domain rules, provider credentials, and event history.
- `webNavigation`: detects top-level browser navigation so Swan can match configured adult-site domains and redirect to the intervention page.
- `https://api.elevenlabs.io/*`: starts user-configured AI phone calls.
- `https://api.twilio.com/*`: sends optional user-configured SMS alerts.

## Reviewer notes

Provide direct setup instructions for reviewers:

1. Install the extension.
2. Open Swan settings from the toolbar icon.
3. Add a recipient phone number.
4. Add ElevenLabs API key, Agent ID, and Agent phone number ID.
5. Optional: add Twilio SMS credentials and enable SMS.
6. Click **Send test alert**.
7. Add a test tracked domain and navigate to it to confirm redirect behavior.

If reviewer-owned provider credentials are not available, explain that call/SMS delivery requires user-managed ElevenLabs/Twilio accounts, but the extension can still be reviewed for settings, local storage, domain rules, event logs, and redirect behavior.

## Submission steps

1. Register or use an existing Chrome Web Store developer account.
2. Create a new item in the Chrome Web Store Developer Dashboard.
3. Upload the WXT Chrome MV3 zip.
4. Complete store listing, privacy, permissions, and distribution fields.
5. Choose **Unlisted** distribution.
6. Submit for review.
7. After approval, smoke-test the install link in a fresh Chrome profile before sharing it.
